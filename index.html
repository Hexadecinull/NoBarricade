Detection Architecture
Six parallel methods.
Any hit gates the page.

Methods run concurrently with early-exit logic — the gate fires the moment the sensitivity threshold is crossed, without waiting for slower methods to finish.

01 — DOM BAIT
Bait Element

A hidden div carrying 35+ ad-targeting class names is injected. uBlock Origin, Adblock Plus, and AdGuard all apply display:none to it via their cosmetic filter rules. The resulting zero dimensions are measured.

02 — NETWORK PROBE
Multi-Network Fetch

Four random URLs are sampled from a pool of 30+ ad network endpoints — Google, AppNexus, Criteo, Taboola, Amazon Ads, Rubicon, and more. All four must succeed for the user to be cleared. Catches DNS-level blockers (Pi-hole, NextDNS, AdGuard Home).

03 — SCRIPT LOAD
Script Tag Test

Two script elements pointing to ad network JS files are injected. Ad blockers intercept them at the network level and fire onerror, which NoBarricade captures. Both must succeed to clear.

04 — CSS ANIMATION
CSS Cosmetic Filter Detection

A CSS animation is applied to a bait element. Ad blockers that apply display:none via cosmetic filters prevent the animation from starting. No animationstart event within the timeout window = blocked. Catches Brave Shields and AdGuard cosmetic filtering.

05 — WINDOW PROPERTIES
Global Property Check

Checks window globals from multiple ad platforms: Google's google_ad_status and googletag, AppNexus's apntag, Taboola's _taboola, and Criteo's criteo_q. Effective when those ad scripts were expected to load.

06 — BRAVE API
Brave Shields Detection

Brave exposes navigator.brave.isBrave() — a Promise that resolves to true in Brave browser with Shields active. Enables direct, reliable detection of Brave's built-in ad blocker without relying on network probes. Opt-in via methods: ['brave'].

Installation
One line. Works everywhere.

No npm, no build step, no server configuration. Just a script tag before your closing </body>.

<script src="https://cdn.jsdelivr.net/gh/hexadecinull/nobarricade@latest/nobarricade.js"></script> 
NoBarricade.init();
NoBarricade.init({
  methods: ['bait', 'fetch', 'script', 'css', 'property'],
  sensitivity:       1,
  checkInterval:    3000,
  detectionTimeout: 850,

  protect: {
    rejectContextMenu:       true,
    rejectKeyboardShortcuts: true,
    lockScroll:              true,
    watchDevTools:           true,
    watchInterval:           750,
  },

  debug: false,
});
NoBarricade.init({
  overlay: {
    title:       'Please Disable Your Ad Blocker',
    message:     'Advertising keeps this site free. Disable your ad blocker to continue.',
    buttonText:  'I\'ve Disabled It — Show Me The Content',
    accentColor: '#e63946',
    brand:       'The Daily Post',
    logo:        '/assets/logo.svg',
    showSteps:   true,
    steps: [
      'Click the ad blocker icon in your browser toolbar.',
      'Select "Pause on this site" or "Disable".',
      'Refresh or click the button below.',
    ],
  },
});
NoBarricade.init({
  onDetected: function () {
    analytics.track('adblock_detected');
  },
  onCleared: function () {
    analytics.track('adblock_cleared');
  },
});
import { useEffect } from 'react';

export default function Layout({ children }) {
  useEffect(function () {
    const nb = require('nobarricade');
    nb.init({ overlay: { brand: 'My Site' } });
    return function () { nb.destroy(); };
  }, []);
  return children;
}

Detection

Six Parallel Methods

DOM bait, multi-network fetch, script load, CSS animation, property check, and Brave API. Early-exit logic fires the gate the moment the threshold is crossed.

30+ Ad Networks

Not Just Google

Fetch probes sample from 30+ endpoints — Google, AppNexus, Criteo, Taboola, Amazon, Rubicon, PubMatic, OpenX, ShareThrough, and more. All must succeed to clear.

DNS-Level Blockers

Pi-hole & NextDNS

DNS-level blockers produce no CSS injection — only network failures. The multi-network fetch and script probes are specifically designed to catch timeout-based blocking.

Gate Integrity

Tamper Resistant

MutationObserver watches for overlay removal or style mutation. Any DevTools-based attempt to delete or hide the overlay is repaired on the next tick with forced inline styles.

Mobile & Desktop

Fully Responsive

The gate card adapts gracefully across all viewport sizes. All detection methods are designed to work identically on desktop, tablet, and mobile browsers.

Verification

Re-verified on Dismiss

When a user clicks "I've disabled it", all detection methods re-run from scratch before the gate is lifted. A shake animation and error message notify them if still blocked.
